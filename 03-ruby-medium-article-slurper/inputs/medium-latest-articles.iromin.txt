This tutorial is a step by step guide to writing a Slack Slash command. A Slack Slash command is a great way for users to interact with a custom application. As the documentation states, it allows users to invoke your app by typing a string into the message composer box. An example from the documentation is shown below:Reference: https://api.slack.com/interactivity/slash-commandsText Summarization Slash Command in ActionLet us first take a look at what we shall be building out.Assuming that our custom Slack Application that contains the Slash Command is installed for a specific Slack Workspace (instructions for which we will provide later in the blog post), the user will invoke a text summarization request ( /summarize command) for a passage of text as follows:On clicking the Send button, this will perform the summarization and return back a response as shown below:How does it work?The solution is architected in the following manner:Slack Slash Command --&gt; Cloud Function --&gt; Vertex AI PaLM APIWe wrap the Vertex AI PaLM Text Bison Model into a single Cloud Function is exposed over a HTTPS endpoint.The Slack Slash Command is configured to invoke the above HTTPS endpoint. It passes the data to be summarized as a JSON object.The Cloud Function receives the data, creates the prompt and invoke the predict method of the Vertex AI PaLM Text Bison model.The response is returned to the Google Cloud Function that wraps the response with Slack specific Block Markdown and returns it back to the Slash Command that renders it inside the Slack Workspace.Setting up the Summarization Slash CommandLet’s setup the whole process now. Follow the step by step instructions.PrerequisitesGoogle Account and Google Cloud Project has been created. If not, please refer to the following SETUP.md file in the repository that provides instructions to setup the Google Cloud project.Slack Account and sample workspace in which you have Administrative rights to create an application.Given the prerequisites, we will be following this sequence:Deploy the Cloud FunctionInstall the Slack ApplicationVerify Happy PathDeploy the Cloud FunctionWe are going to be writing a Cloud Function that will invoke the PaLM Bison Text Model for us. The source code for the same is shown below:https://medium.com/media/cc6321c7a6b07425a972dfc55c723d72/hrefLet’s go through the Cloud Function in bried:It has a summarizeText method that extracts out the text that has been passed to it via the POST method.Next, we initialize the Vertex AI module and then call the predict method with the parameters. You can tweak them if you’d like. You will notice that we need two parameters : GCP_PROJECT and GCP_REGION to initialize Vertex AI and this will be read from the environment variables that are passed to it (more on this when we deploy the Cloud Function).The response is then packaged in Slack Markdown to format it for the Slack Message in the Workspace.TODO for the reader: A good practice is to validate the Security Token that is passed to you by Slack.
