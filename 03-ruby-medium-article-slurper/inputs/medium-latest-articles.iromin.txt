            One of my repositories that has seen good traction is the Google Cloud GenAI Application templates. These are a set of application templates that use Google Cloud services (Cloud Functions , Cloud Run) to interact with Google Cloud Vertex AI models like Text Bison, Chat Bison and more.There are several application templates that range from Cloud Functions that envelop the Gen AI models (Text Bison, Code Bison, Chat Bison) to Slack Application Integration, Google Chrome Extensions and more. You can check out the repository over here.GitHub - rominirani/genai-apptemplates-googlecloud: Sample Applications to kickstart Generative AI Application Development on Google Cloud.All the code samples provided so far were with Python. I do not have any language as my favourite and neither am I great at any of the languages but if there is one thing that I have learnt, it is to try and provide examples across multiple languages to cover as many developers out there.Given that, I am started a move to migrate some of these code samples to another language and this time it is Java. What you see in this blog is to write a Google Cloud Function in Java that invokes the Chat Bison PaLM2 Model hosted in Vertex AI and use the an open source framework langchain4j that is based on the popular langchain framework.The entire source code with instructions, should you want to try that out is available over here:genai-apptemplates-googlecloud/chat-predict-cloudfunction-java at main · rominirani/genai-apptemplates-googlecloudLet’s go through a step by step process of deploying and running this Google Cloud Function.Pre-requisitesYou will need to have a Google Cloud Project with a set of Google Cloud services enabled for that project. Please refer to the SETUP.md file that contains step by step instructions to enable these services.Java ProjectFirst up is the pom.xml file that will contain the necessary dependencies for langchain4j framework and other associated utilities. The listing is given below:Maven dependencies — pom.xmlhttps://medium.com/media/7b544a0a1982720b38253d401ec726bb/hrefA few points about the pom.xml file:The langchain4j dependencies, which also includes the langchain4j-vertex-ai since we are going to be integrating with the Vertex AI APIs that talk to the hosted core foundational models.Since the code does some basic logging, I have added the logback dependencies.In addition to the above, you will see the functions-framework-api since Java Cloud Functions makes use of that.Source Code : PredictChatFunction.javaThe source code for the same is given below:https://medium.com/media/e2b6cfe32ab67b2356bbae635dd4a88b/hrefNote the following about the Google Cloud Function source code above:The Cloud Function is invoked with a JSON body passed to it, in the following format:{"prompt":"&lt;YOUR_PROMPT_HERE"}2. Lines 38–39 extract out the JSON body above from the request. and then till line 46, we are extracting out the prompt variable.3. If the prompt is provided, we instantiate the VertextAiChatModel class with various properties include the Project Id and Location along with the model that we are invoking here (Chat Bison) and the model parameters like TopP, TopK, Temperature, etc.4. Finally we invoke the generate method of the model instance and extract out the model response to our prompt.Deploying the Cloud FunctionFirst setup the following environment variables in your setup. For e.g. if you are in Cloud Shell, set the two variables as shown below:export GCP_PROJECT='&lt;Your GCP Project Id&gt;'  export GCP_REGION='us-central1'     Assuming that you have created the folder structure as mentioned in the Github folder for this application in your local setup (Cloud Shell), follow these steps:Go to the root folder of this project where the pom.xml file is present.Provide the following command:gcloud functions deploy PredictChatFunction \--gen2 \--runtime=java11 \--region=$GCP_REGION \--source=. \--entry-point=gcfv2.PredictChatFunction \--trigger-http \--set-env-vars=GCP_PROJECT=$GCP_PROJECT,GCP_REGION=$GCP_REGION \--allow-unauthenticatedInvoking the Cloud FunctionSince this Cloud Function is deployed with a HTTP trigger, you can directly invoke it. A sample call is shown below:curl -m 70 -X POST https://$GCP_REGION-$GCP_PROJECT.cloudfunctions.net/PredictChatFunction \-H "Content-Type: application/json" \-d '{  "prompt": "what is langchain4j?"}'You can replace the prompt above in the sample with your own prompt. On successful execution, you should get back the response to your prompt.ConclusionThis concludes the post on migrating one of the Cloud functions from Python to Java. This Cloud Function uses langchain4j framework, which is currently under development, so do expect a few changes to it in the coming months.Do check out the entire repository for samples on Gen AI + Google Cloud that will speed up your process of deploying some of this functionality to Google Cloud.Do give me feedback on some of the samples that you would like to see.Integrating langchain4j and PaLM 2 Chat Bison Model was originally published in Google Cloud - Community on Medium, where people are continuing the conversation by highlighting and responding to this story.
